KMP/Compose Multiplatform Member App · afgeleid uit de daadwerkelijke broncode (mobile/)

4 Klassendiagram (eigen klassen)

Auth- en core-slice: ViewModel → Repository → API → HttpClient, met secure storage en biometrie. De contributie- en profiel-slices volgen exact hetzelfde patroon.

classDiagram
  class LoginViewModel {
    +state: StateFlow~LoginUiState~
    +login()
    +onEmailChange(value)
    +onPasswordChange(value)
  }
  class LoginUiState {
    +email: String
    +password: String
    +isLoading: Boolean
    +error: String?
    +loggedIn: Boolean
  }
  class AuthRepository {
    +currentUser: StateFlow~UserDto~
    +isLoggedIn: Boolean
    +login(email, password) Result
    +logout()
    +validateSession() SessionStatus
    +authToken() String?
    +organisationSubdomain() String?
  }
  class AuthApi {
    +login(email, password) TokenResponse
    +me(token) UserDto
    +logout(token)
  }
  class SessionStorage {
    +save(token, user)
    +token() String?
    +user() UserDto?
    +hasSession() Boolean
    +clear()
  }
  class BiometricAuthenticator {
    +canAuthenticate() Boolean
    +authenticate(title, subtitle) BiometricResult
  }
  class UserDto {
    +id: Int
    +email: String
    +name: String?
    +status: String?
    +organisation: OrganisationDto?
  }

  LoginViewModel --> LoginUiState : exposes
  LoginViewModel --> AuthRepository
  AuthRepository --> AuthApi
  AuthRepository --> SessionStorage
  AuthRepository --> UserDto : currentUser
  AuthApi --> UserDto : returns

5 Sequencediagram — MVVM (login)

View → ViewModel → Repository → API, met state-updates via StateFlow (verplicht: min. 1 MVVM-sequence).

sequenceDiagram
  actor U as Member
  participant V as LoginScreen
  participant VM as LoginViewModel
  participant R as AuthRepository
  participant A as AuthApi
  U->>V: e-mail + wachtwoord, tik "Sign in"
  V->>VM: login()
  VM->>VM: state = isLoading
  VM->>R: login(email, password)
  R->>A: login(email, password)
  A-->>R: TokenResponse
  R->>R: SessionStorage.save(token, user)
  R-->>VM: Result.success
  VM->>VM: state = loggedIn
  VM-->>V: StateFlow update
  V->>U: navigatie naar Dashboard

6 Sequencediagram — app ↔ backend (token-validatie)

Bij het ontgrendelen wordt het opgeslagen token gevalideerd tegen de backend (verplicht: min. 1 app↔backend-sequence).

sequenceDiagram
  participant App as UnlockScreen
  participant R as AuthRepository
  participant A as AuthApi
  participant B as Backend (api.aidatim.nl)
  App->>R: validateSession()
  R->>A: me(token)
  A->>B: GET /api/auth/me
Authorization: Bearer
  alt token geldig
    B-->>A: 200 · UserDto
    A-->>R: UserDto
    R->>R: profiel verversen + opslaan
    R-->>App: VALID → Dashboard
  else token verlopen/ongeldig
    B-->>A: 401 Unauthorized
    A-->>R: UnauthorizedException
    R->>R: SessionStorage.clear()
    R-->>App: INVALID → Login
  end
